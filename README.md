Organisation Description
The New Mahlalela is a non-profit organisation founded in 2024 in Matatiele, Eastern Cape, South Africa.  It was created to address the post-matric crisis facing South African youth. Each year hundreds of thousands of young people finish Grade 12 but only a small fraction gain access to universities or colleges. Those who do not are often left feeling isolated, ashamed, and uncertain about their future. What started as a small weekly meeting in local churches has grown to serve three communities, with over 50 trained mentors and 200 active participants. The organisation uses donated community spaces such as churches, schools, and halls, and relies entirely on volunteers and small local donations.

Reasons for Choosing this Organisation
 The New Mahlalela addresses a critical and often overlooked challenge facing South African youth. While other organisations offer career guidance or mental health services serparately, very few combine both in a way that is rooted in local community knowledge and designed for young people with limited resources. The organisation currently relies entirely on face-to-face meetings, word-of-mouth, and volunteer networks, which limits its reach. Building a website will solve this by expanding access, increasing visibility to donors and partners, and streamlining administration.

Improvements to Part 2 Website and SEO Strategies
SEO stands for Search Engine Optimisation. It means making a website easier for search engines like Google to find and understand. The following SEO improvements were made to the website for The New Mahlalela.
Page Titles
Every page was given a clear and descriptive title tag inside the head section. For example the home page title was changed from just "Home" to "Home | The New Mahlalela". A good page title helps search engines understand what each page is about and also shows up in the browser tab so users know which page they are on.
Alt Text on Images
All images on the website were given proper alt text descriptions. Alt text is a short description of what an image shows. It helps search engines understand the image because they cannot see pictures. It also helps people who use screen readers because the screen reader reads the alt text out loud to them. For example the team member images were given alt text such as "Photo of Finance Manager Lily Smith" instead of just "logo" or leaving it blank.
Proper Use of Headings
Each page was given one main h1 heading that describes the page. Subheadings use h2 and h3 tags in the correct order. Search engines use headings to understand the structure and topic of a page. Having one clear h1 heading and using h2 and h3 in the right order helps search engines rank the page correctly.
Meaningful Link Text
All navigation links were given clear and meaningful names such as "Support and Pathways" and "Community and Stories" instead of vague words. The external link to the SADAG website on the Support and Pathways page was also given a clear label so users and search engines know where the link goes.
Mobile Friendly Design
The website was designed to work on phones and tablets using CSS. The viewport meta tag was added to the head section of every page. This tag tells the browser to adjust the width of the page to fit the screen size of the device being used. Google gives higher search rankings to websites that work well on mobile devices because most people browse the internet on their phones.

Steps on Hosting a Website
Hosting a website means putting your website files on the internet so that anyone can open them in a browser. The website for The New Mahlalela was hosted using GitHub Pages which is a free hosting service. Below are the steps that were followed to host the website.
Step 1: Create a GitHub Account
Go to www.github.com and click Sign Up. Fill in your email address, create a username and a password, then click Create Account. GitHub is a free platform where developers store and share their code.
Step 2: Create a New Repository
Once logged in, click the green New button to create a new repository. A repository is like a folder on GitHub where your website files will be stored. Give the repository a name, make sure it is set to Public, then click Create Repository.
Step 3: Upload Your Website Files
Inside the new repository, click Add File and then Upload Files. Drag and drop all your website files into the upload area. This includes all your HTML pages, your CSS folder, your JavaScript folder, and your images folder. Click Commit Changes to save the files.
Step 4: Enable GitHub Pages
Click on the Settings tab inside your repository. Scroll down until you see the Pages section on the left side menu and click on it. Under the Branch option, select Main and then click Save. GitHub will now build your website and give you a live link.
Step 5: Open Your Live Website
After a few minutes, go back to the Pages section in Settings and you will see a green message with your website link. Click the link to open your website in a browser. The website for The New Mahlalela is now live and anyone in the world can visit it.
Key Functionalities of the Website
The following are the main features and functionalities that were built into the website for The New Mahlalela.
Navigation Menu
Every page has a navigation bar inside the header that links to all six pages of the website. This allows users to move between pages easily without having to use the back button. The navigation bar appears at the top of every page.
Contact Form with Validation
The Contact page has a form where users can type their name, email address, and a message to send to the organisation. JavaScript checks that all fields are filled in before the form is submitted. If a field is empty, a red error message appears under it. If everything is filled in correctly, a green success message appears to confirm the message was sent.
Google Maps Embed
The Contact page includes an embedded Google Map that shows the location of Matatiele, Eastern Cape. This helps visitors who are not familiar with the area to find where the organisation is based.
Workshop Registration Form
The Support and Pathways page has a form that allows young people to register for upcoming workshops. The form checks that the name and email fields are filled in and shows a success message when registration is complete.
Mentor Application Form
The Get Involved page has a form where community members can apply to become mentors. The form uses JavaScript validation and shows a success message when the application is submitted correctly.
Donation Impact Calculator
The Get Involved page has a calculator where a visitor can type in a rand amount and see how many young people that donation could help attend a workshop. For example typing in R200 would show that 10 young people could be helped. This encourages people to donate by showing them the real impact of their contribution.
Expandable Story Cards
The Community and Stories page shows three stories from young people in Matatiele. Each story is hidden at first. When a user clicks on the story heading, the full story appears below it. Clicking again hides it. This makes the page look clean and organised while still showing all the content.
Team Member Bio Toggle
The About page displays four team member cards. Each card shows the team member photo, job title, and name. When a user clicks on a card, a short bio about that person appears below their name. Clicking again hides the bio. This keeps the page tidy while still giving visitors information about the team.
Live Clock
The Community and Stories page displays the current time in hours, minutes and seconds. The clock updates every second automatically using JavaScript. This adds an interactive element to the page and shows that JavaScript is working on the website.
Back to Top Button
A back to top button appears on every page when the user scrolls down more than 200 pixels. The button is fixed in the bottom right corner of the screen. Clicking it smoothly scrolls the page back to the top. This is useful on pages that have a lot of content such as the Community and Stories page.
